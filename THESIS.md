## 1. Εισαγωγή

Η εφαρμογή CookMe αναπτύχθηκε στο πλαίσιο της διπλωματικής εργασίας στο Τμήμα Επιστήμης Υπολογιστών του Πανεπιστημίου Κρήτης. Σκοπός της εργασίας είναι η δημιουργία μιας ολοκληρωμένης διαδικτυακής πλατφόρμας που επιτρέπει στους χρήστες να ανακαλύπτουν συνταγές μαγειρικής και να παραγγέλνουν τα απαιτούμενα υλικά από το πλησιέστερο ή οικονομικότερο σούπερ μάρκετ στην περιοχή τους.

Το έγγραφο αυτό παρουσιάζει αναλυτικά τον σκοπό ανάπτυξης του συστήματος, τα χαρακτηριστικά του, τις απαιτήσεις, τη σχεδίαση, την αρχιτεκτονική και τα αποτελέσματα αξιολόγησής του. Απευθύνεται στα εμπλεκόμενα μέλη της επιτροπής αξιολόγησης καθώς και σε κάθε ενδιαφερόμενο που επιθυμεί να κατανοήσει τη λειτουργία και την υλοποίηση του συστήματος.

### 1.1 Γενική Περιγραφή

Το CookMe είναι μια διαδικτυακή εφαρμογή (web application) δύο επιπέδων: ένα frontend (COOKME_UI) βασισμένο σε Vue 3 και ένα backend REST API (COOKME_rest-api) γραμμένο σε Go. Η επικοινωνία τους γίνεται μέσω HTTP/JSON.

Ανάλογα με τον τύπο του χρήστη που κάνει είσοδο στο σύστημα, παρουσιάζεται η αντίστοιχη διεπαφή:

- **Ανώνυμος χρήστης (Guest):** Μπορεί να περιηγηθεί στις συνταγές και να δει λεπτομέρειες, αλλά δεν μπορεί να κάνει παραγγελία ή να δει τιμές ανά σούπερ μάρκετ με βάση τοποθεσία.
- **Εγγεγραμμένος χρήστης (User):** Έχει πρόσβαση σε όλες τις λειτουργίες: αναζήτηση συνταγών, checkout με σύγκριση τιμών σούπερ μάρκετ, παραγγελία υλικών, αγαπημένες συνταγές, κριτικές και προφίλ.
- **Διαχειριστής (Admin):** Έχει πρόσβαση στο πίνακα διαχείρισης για παρακολούθηση χρηστών και παραγγελιών.

### 1.2 Σκοπός Συστήματος

Το σύστημα CookMe αντιμετωπίζει ένα κοινό πρόβλημα: ο χρήστης βρίσκει μια συνταγή που θέλει να μαγειρέψει, αλλά δεν γνωρίζει πού να αγοράσει τα υλικά και σε ποια τιμή. Το CookMe λύνει αυτό με αυτοματοποιημένη:

1. **Ανακάλυψη συνταγών** — αναζήτηση με λέξεις-κλειδιά ή φίλτρα κουζίνας.
2. **Σύγκριση τιμών** — εύρεση του φθηνότερου ή πλησιέστερου σούπερ μάρκετ (AB, Lidl, Σκλαβενίτης, MyMarket) με βάση τη γεωγραφική θέση του χρήστη.
3. **Παραγγελία** — ολοκλήρωση αγοράς υλικών απευθείας μέσα από την εφαρμογή με επιλογή χρόνου παράδοσης ή παραλαβής.

## 2. Σενάρια Χρήσης και Τύποι Χρηστών

### Τύποι Χρηστών

| Τύπος | Περιγραφή |
|---|---|
| **Guest** | Ανώνυμος επισκέπτης χωρίς σύνδεση |
| **User** | Εγγεγραμμένος χρήστης |
| **Admin** | Διαχειριστής συστήματος |

### Σενάριο 1 — Ανακάλυψη Συνταγής (Guest / User)

1. Ο χρήστης ανοίγει την εφαρμογή και βλέπει τη λίστα συνταγών.
2. Φιλτράρει βάσει κουζίνας (π.χ. Ελληνική, Ιταλική) ή επιλέγει κατηγορία (Pizza, Fish, Pork, Cocktail, Quick).
3. Πληκτρολογεί στο πεδίο αναζήτησης (Elasticsearch full-text search).
4. Κάνει κλικ σε μια συνταγή για να δει τα υλικά, τον χρόνο μαγειρέματος, την κουζίνα και τη βαθμολογία.

### Σενάριο 2 — Εγγραφή Χρήστη

1. Ο χρήστης κάνει κλικ στο "Register" στη σελίδα Login.
2. Συμπληρώνει: Username, Email, Password, Διεύθυνση.
3. Το σύστημα αποθηκεύει τον λογαριασμό.
4. Ο χρήστης ανακατευθύνεται στην αρχική σελίδα.

### Σενάριο 3 — Σύνδεση Χρήστη

1. Ο χρήστης εισάγει Email και Password.
2. Το σύστημα επαληθεύει τα στοιχεία.
3. Αν ο χρήστης είναι `admin` ανακατευθύνεται στο admin panel, αλλιώς στην αρχική σελίδα.
4. Η ταυτότητα αποθηκεύεται στο localStorage για persistent login.

### Σενάριο 4 — Παραγγελία Υλικών (User)

1. Ο χρήστης επιλέγει μια συνταγή και κάνει κλικ "Checkout".
2. Το σύστημα εντοπίζει τα διαθέσιμα σούπερ μάρκετ εντός 5χλμ από τον χρήστη.
3. Αυτόματα επιλέγεται το **φθηνότερο** σούπερ μάρκετ (ή ο χρήστης αλλάζει σε "πλησιέστερο").
4. Εμφανίζεται λίστα υλικών με τιμές. Ο χρήστης επιλέγει ποια θέλει.
5. Συμπληρώνει στοιχεία πληρωμής και επιλέγει παράδοση τώρα ή σε συγκεκριμένη ώρα.
6. Η παραγγελία αποθηκεύεται με κατάσταση "In Progress".

### Σενάριο 5 — Αγαπημένες Συνταγές & Κριτικές (User)

1. Ο χρήστης προσθέτει συνταγές στα αγαπημένα μέσω του κουμπιού ♥.
2. Μπορεί να αφήσει βαθμολογία (1–5 αστέρια) και σχόλιο σε οποιαδήποτε συνταγή.
3. Το ιστορικό παραγγελιών και αγαπημένων εμφανίζεται στη σελίδα προφίλ.

### Σενάριο 6 — Διαχείριση Συστήματος (Admin)

1. Ο admin συνδέεται και ανακατευθύνεται αυτόματα στο admin panel.
2. Μπορεί να δει όλους τους χρήστες και να διαγράψει λογαριασμούς.
3. Μπορεί να δει και να διαχειριστεί όλες τις παραγγελίες.

| F-08 | Σύγκριση τιμών υλικών μεταξύ σούπερ μάρκετ |

| F-13 | Admin panel για διαχείριση χρηστών και παραγγελιών |

### 3.2 Μη-Λειτουργικές Απαιτήσεις

| Κατηγορία | Απαίτηση |
|---|---|
| **Απόδοση** | Η λήψη λίστας συνταγών να ολοκληρώνεται σε < 2 δευτερόλεπτα |
| **Διαθεσιμότητα** | Το σύστημα να λειτουργεί αδιάλειπτα κατά τη διάρκεια χρήσης |
| **Ασφάλεια** | Κωδικοί χρηστών αποθηκευμένοι με hashing. CORS ενεργοποιημένο στο API |
| **Χρηστικότητα** | Responsive διεπαφή με σαφή πλοήγηση, φόρτωση με spinner, ειδοποιήσεις σφαλμάτων |
| **Φορητότητα** | Εκτέλεση σε οποιοδήποτε λειτουργικό σύστημα μέσω Docker |
| **Επεκτασιμότητα** | Hexagonal architecture (ports & adapters) στο backend για εύκολη αντικατάσταση εξαρτήσεων |
| **Διαλειτουργικότητα** | REST API με JSON για επικοινωνία frontend/backend. Ενσωμάτωση Google Maps API |
| **Συντηρησιμότητα** | Διαχωρισμός σε layers (handlers / core / repositories) |

### 3.3 Περιορισμοί

- Η λειτουργία εύρεσης σούπερ μάρκετ απαιτεί έγκυρη γεωγραφική θέση χρήστη (latitude/longitude).
- Απαιτείται έγκυρο Google Maps API key με ενεργοποιημένα Places API, Geocoding API και Distance Matrix API.
- Η αναζήτηση μέσω Elasticsearch απαιτεί ότι η υπηρεσία Elasticsearch είναι διαθέσιμη κατά την εκκίνηση του API.
- Τα δεδομένα τιμών σούπερ μάρκετ είναι στατικά (αρχεία .txt) και δεν ενημερώνονται αυτόματα.

### 5.2 Αρχιτεκτονική Λογισμικού Backend

Το backend ακολουθεί **Hexagonal Architecture** (Ports & Adapters):

```
HTTP Request
     │
     ▼
┌──────────┐      ┌──────────┐      ┌─────────────────┐
│ Handler  │ ───> │   Core   │ ───> │   Repository    │
│(HTTP I/O)│      │(Business │      │(DB / Elastic /  │
│          │      │  Logic)  │      │  Google Maps)   │
└──────────┘      └──────────┘      └─────────────────┘
     │                  │                    │
  fiber.Ctx          ports.go            gorm / ES client
```

- **Handlers** (`handlers/`): Αποκωδικοποιούν HTTP request, καλούν το Core, επιστρέφουν JSON response.
- **Core** (`core/`): Περιέχει την επιχειρησιακή λογική (σύγκριση τιμών, εύρεση αγοράς, indexing ES).
- **Repositories** (`repositories/`): Αλληλεπιδρούν με PostgreSQL μέσω GORM.
- **Ports** (`ports/`): Go interfaces που ορίζουν τις συμβάσεις μεταξύ layers.

### 5.3 Λογική Σύγκρισης Τιμών Σούπερ Μάρκετ

```
Χρήστης κάνει checkout
         │
         ▼
findAllAvailableMarkets(lat, lon)
   └── Google Places API → σούπερ μάρκετ εντός 5km
   └── chooseMarkets() → φιλτράρισμα σε AB/Lidl/Sklavenitis/MyMarket
   └── Distance Matrix API → υπολογισμός απόστασης για κάθε αγορά
         │
         ▼
compareMarketPrices(recipeId, markets)
   └── Για κάθε υλικό της συνταγής:
       └── Αναζήτηση τιμής στον πίνακα market_ingredients
       └── Άθροισμα ανά σούπερ μάρκετ
   └── Επιλογή φθηνότερου → επιστροφή Market
         │
         ▼
convertToMarketIngredients(recipeId, market)
   └── Επιστροφή λίστας {name, price} για εμφάνιση στο UI
```

## 8. Οδηγίες Χρήσεως του Συστήματος

### 8.1 Ανώνυμος Χρήστης (Guest)

1. Ανοίξτε `http://localhost:8080` στον browser σας.
2. Αναζητήστε συνταγές χρησιμοποιώντας τη μπάρα αναζήτησης ή τα φίλτρα αριστερά.
3. Κάντε κλικ σε μια συνταγή για να δείτε τα υλικά και τον χρόνο μαγειρέματος.
4. Για παραγγελία απαιτείται σύνδεση.

### 8.2 Εγγεγραμμένος Χρήστης (User)

**Εγγραφή:**
1. Κάντε κλικ "Login" στο header.
2. Επιλέξτε την καρτέλα "Register".
3. Συμπληρώστε: Username, Email, Password, Διεύθυνση.
4. Κάντε κλικ "Sign up".

**Παραγγελία υλικών:**
1. Επιλέξτε συνταγή από την αρχική σελίδα.
2. Κάντε κλικ στο κουμπί Checkout.
3. Επιλέξτε σούπερ μάρκετ (αυτόματα επιλέγεται το φθηνότερο) ή αλλάξτε σε "Distance" για το πλησιέστερο.
4. Επιλέξτε τα υλικά που θέλετε (προεπιλεγμένα όλα).
5. Κάντε κλικ "Checkout", συμπληρώστε τα στοιχεία κάρτας.
6. Επιλέξτε "Delivery Now" ή προγραμματίστε παράδοση.
7. Κάντε κλικ "Order".

**Αγαπημένα:**
- Κάντε κλικ ♥ σε οποιαδήποτε συνταγή για να την προσθέσετε στα αγαπημένα σας.
- Τα αγαπημένα εμφανίζονται στη σελίδα Προφίλ.

### 8.3 Διαχειριστής (Admin)

1. Συνδεθείτε με λογαριασμό τύπου `admin`.
2. Ανακατευθύνεστε αυτόματα στο Admin Panel.
3. Δείτε όλους τους χρήστες — κάντε κλικ Διαγραφή για να αφαιρέσετε λογαριασμό.
4. Δείτε όλες τις παραγγελίες και την κατάστασή τους.

## 9. Επίλογος

### 9.1 Συμπεράσματα

Το CookMe υλοποιήθηκε επιτυχώς ως ολοκληρωμένη πλατφόρμα συνταγών και παραγγελίας υλικών. Συνδυάζει σύγχρονες τεχνολογίες backend (Go, Fiber, GORM, Elasticsearch) με ένα πλούσιο frontend (Vue 3, PrimeVue) και ενσωματώνει εξωτερικές υπηρεσίες (Google Maps) για τη γεωγραφική εντόπιση σούπερ μάρκετ και τη σύγκριση τιμών.

### 9.2 Αδυναμίες

- Τα δεδομένα τιμών σούπερ μάρκετ είναι στατικά και δεν ενημερώνονται αυτόματα.
- Οι κωδικοί χρηστών αποθηκεύονται χωρίς hashing — απαιτείται προσθήκη bcrypt.
- Δεν υπάρχει email επιβεβαίωσης κατά την εγγραφή.
- Χρήστες που εγγράφονται χωρίς έγκυρες συντεταγμένες δεν έχουν πρόσβαση στην εύρεση πλησιέστερης αγοράς.

### 9.3 Μελλοντικές Επεκτάσεις

- Ενσωμάτωση πραγματικών APIs σούπερ μάρκετ για live τιμές.
- Προσθήκη hashing κωδικών (bcrypt).
- Δυνατότητα ενημέρωσης τοποθεσίας χρήστη από την εφαρμογή.
- Mobile application (React Native ή Flutter).
- Σύστημα ειδοποιήσεων για κατάσταση παραγγελίας.
- Recommendation engine βασισμένο στις προτιμήσεις χρήστη.
